**Шаг 1. Определите бизнес-задачу**

- Это **отправная точка** — без цели невозможно выбрать верные действия.
- Примеры бизнес-задач:
    1. Выход из прямой конкуренции.
    2. Запуск продукта на рынок.
    3. Увеличение размера рынка.
    4. Рост ценности продукта (первый приоритет и остальные способы).
    5. Получение нового канала привлечения.
    6. Масштабирование внутри сегмента.
    7. Рост конверсии.
    8. Рост среднего чека.
    9. Рост возвращаемости.
    10. Снижение оттока.

**Пример для AI-Терапевта:**

Если задача — **увеличить возвращаемость**, то работа будет строиться вокруг удерживающих механик, а не каналов привлечения.

### **Шаг 2. Сформируйте гипотезы механик**

- Для каждой бизнес-задачи есть **список приоритетных механик** — они подсказывают, где в графе работ можно найти решение.
- Ваша задача на этом этапе — **не выбрать одну механику**, а составить **набор гипотез**, какие механики могут сработать именно под вашу цель.
- Гипотезы формируются «на черновик» — без глубокой проверки, но с привязкой к реальным шагам графа.

**Как формулировать гипотезу механики:**

> «Если применить механику [название], то сможем повлиять на [метрика], потому что [логика воздействия]».
> 

### **Шаг 3. Получите сырьё для проверки гипотез**

- **Цель этапа** — собрать фактическую информацию, необходимую для проверки гипотез механик, сформированных на шаге 2.
- **Сырьё** — данные о сегментах, их работах, барьерах, критериях выбора и уровне спроса.
- **Как получать:**
    1. **Качественные исследования** — глубинные и решенческие интервью, наблюдение за поведением.
    2. **Внешние источники** — аналитические отчёты, данные конкурентов, публикации.
    3. **Экспертные консультации** — общение с отраслевыми специалистами.
    4. **Количественные методы** — опросы, тестовые офферы, A/B-тесты.
- **Результат шага:**
    - Чёткое понимание, в каких точках графа есть потенциал для реализации идеи.
    - Данные, на основе которых можно переходить к формулировке продуктовых гипотез.

### **Шаг 4. Сформулируйте продуктовые гипотезы**

- **Задача этапа** — превратить проверенные гипотезы механик в конкретные продуктовые решения.
- На входе:
    - набор механик, которые прошли предварительную проверку,
    - данные и инсайты из исследований,
    - граф работ с отмеченными точками воздействия.
- На выходе:
    - список продуктовых гипотез, готовых к приоритизации.

**Как формулировать гипотезу:**

> «Если мы [действие], то сможем [изменение метрики], потому что [логика воздействия и подтверждение данными]».
> 

### **Шаг 5. Приоритизируйте гипотезы по RICE**

- **Задача этапа** — выбрать, какие продуктовые гипотезы запускать первыми, чтобы максимально быстро получить бизнес-результат.
- Метод **RICE**:
    1. **R — Reach** — охват: сколько пользователей затронет гипотеза.
    2. **I — Impact** — влияние на целевую метрику в рамках выбранной бизнес-задачи.
    3. **C — Confidence** — уверенность в гипотезе, подтверждённая данными из шага 3.
    4. **E — Effort** — затраты на реализацию (чем меньше, тем лучше).
- **Результат шага:**
    - Таблица гипотез с оценками по RICE.
    - Отобранный приоритетный пул для тестирования.

---

### **Сакральный ответ «как делать продукт»**

- **Две базовые механики, которые работают на горизонте десятков лет:**
    1. **Убийство работ** — устранение необходимости выполнять определённую работу.
    2. **Выход на более высокий уровень работы** — решение более важной, стратегической задачи.
- **Почему это главное:**
    - Остальные механики часто являются производными от этих двух.
    - Именно они обеспечивают переход продукта на более высокую ценность для пользователя и более сильные позиции на рынке.
